Setting up Environment #back to top

Pod is built on the Javascript ecosystem. Hence make sure you have the following environment setup.

  • * Node.js 8.x+ & NPM 5.6+ : https://nodejs.org/en/
  • * Angular CLI 6+ : https://cli.angular.io/
  • * Firebase CLI

Setting up Firebase CLI

Assuming you have already installed npm above
  1. Open up terminal / Command prompt
  2. Type in npm install -g firebase-tools
    You should now have a globally available firebase command available from any terminal window on your machine.
  3. Once you've installed the Firebase CLI, sign in using your Google account: firebase login
This command connects your local machine to your Firebase account and grants access to your projects. To test that authentication worked, you can run firebase list to see a list of all of your Firebase projects. None ? Don't worry we will create a project in next step

Setting up Firebase #back to top

  1. Create a Firebase project at https://console.firebase.google.com/
  2. Let's enable the following services : Important
    1. Click on Authentication in the Left pane. Select Sign in Method Tabs and enable Email/Password
    2. Click on Storage and enable it. Opt for the default rules
    3. Click on Database and select the Test rules (We will be uploading the rules later)
  3. Select the project and then Click on the Settings Icon from the left sidebar
  4. Select Project Settings
  5. Keep a note of the Project ID
  6. Paste it in .firebaserc 
    {
    "projects": {
        "default": "paste-here"
    }
}
  7. Get back to the firebase console and click on Cloud Messaging
  8. Copy Sender ID (For Push Notifications)
  9. Paste it in app/src/firebase-messaging-sw.js 
    importScripts('https://www.gstatic.com/firebasejs/3.9.0/firebase-app.js');
importScripts('https://www.gstatic.com/firebasejs/3.9.0/firebase-messaging.js');


firebase.initializeApp({
    'messagingSenderId': 'paste-here'
});

const messaging = firebase.messaging();
  10. Paste it in app/src/manifest.json 
    {
    "short_name": "Pod",
    "name": "Pod | Project Management",
    "start_url": "/?utm_source=homescreen",
    "gcm_sender_id": "paste-here"
}

Now as we have the project zip extracted and environment setup, Let us install all the project dependencies

Build and Deployment #back to top

Cloud Functions (FaaS)

Cloud Functions for Firebase lets you automatically run backend code in response to events triggered by Firebase features and HTTPS requests. Your code is stored in Google's cloud and runs in a managed environment. There's no need to manage and scale your own servers. This section guides to deploy the functions onto your firebase project

  1. Open up Firebase project console https://console.firebase.google.com
  2. Click on the Settings Icon from the left sidebar
  3. Select Project Settings
  4. Click on the Service accounts tab
  5. Select Node.js and Click on Generate new private key button
  6. Rename to key.json and paste it to functions/keys sub-directory
  7. Open up a console window (Terminal)
  8. Navigate to functions directory
  9. npm run deploy or yarn deploy
  10. Wait till the deployment process is complete. This may take some time. You may cross check the functions at https://console.firebase.google.com/
  11. Take a note of the *API Base URL
Did you know ?

After you write and deploy a function, Google's servers begin to manage the function immediately. You can fire the function directly with an HTTP request, or, in the case of background functions, Google's servers will listen for events and run the function when it is triggered. As the load increases or decreases, Google responds by rapidly scaling the number of virtual server instances needed to run your function. Each function runs in isolation, in its own environment with its own configuration.

Web Application (Angular 6)

This section describes techniques for deploying the POD Angular application to firebase hosting. You may also deploy the application to a custom server if you wish, which is not covered in this guide.

  1. Open up Firebase project console https://console.firebase.google.com
  2. Select Project Overview from the left sidebar
  3. Click on + Add App button and select Web
  4. Copy the snippet 
    // Initialize Firebase
var config = {
    apiKey: "xxxxxxxxxxxxxxxxxxxxxxxx",
    authDomain: "xxxx-xxxx.firebaseapp.com",
    databaseURL: "https://xxxx-xxxxx.firebaseio.com",
    projectId: "xxxx-xxxxx",
    storageBucket: "xxxx-xxxxx.appspot.com",
    messagingSenderId: "xxxxxxxxxxxxxxx"
};
  5. Replace the keys in app/src/environment/environment.ts and app/src/environment/environment.prod.ts 
                                                export const environment = {
                                                production: false,
                                                firebase: {
                                                  apiKey: '',
                                                  authDomain: '',
                                                  databaseURL: '',
                                                  projectId: '',
                                                  storageBucket: '',
                                                  messagingSenderId: ''
                                                },
                                                cloud : {
                                                  apiUrl : '*API Base URL copied in the previous step | Eg : https://us-central1-xxxx-xxxx.cloudfunctions.net/'
                                                }
                                              };
  6. Open up a console window (Terminal)
  7. npm run deploy or yarn deploy
  8. Wait till the deployment process is complete. This may take some time. You will be provided with the App URL at the end
Did you know ? The --prod meta-flag engages the following optimization features.
  1. Ahead-of-Time (AOT) Compilation: pre-compiles Angular component templates.
  2. Production mode: deploys the production environment which enables production mode.
  3. Bundling: concatenates your many application and library files into a few bundles.
  4. Minification: removes excess whitespace, comments, and optional tokens.
  5. Uglification: rewrites code to use short, cryptic variable and function names.
  6. Dead code elimination: removes unreferenced modules and much unused code.
